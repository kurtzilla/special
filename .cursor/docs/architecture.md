# Engine Blueprint: Isometric C# Engine

## Current Status

- Architecture: DOD (Data Oriented Design)
- Target: High-performance, moddable, server-authoritative

## Component Pattern (The ECS/DOD Hybrid)

- **Entity:** Typed handle `Special.Engine.Ecs.Entity` — packed **index (20 bits) + generation (12 bits)** for stale-safe reuse; minted only via **`Registry`** (not a bare `int` in public API).
- **Component:** A `readonly struct` or plain `struct` containing only data (e.g., `Position`, `Velocity`, **`Transform`** with **`CurrentPosition`** / **`PreviousPosition`** for interpolation, `SpriteRenderer`).
- **Component storage:** Per-type **`ComponentPool<T>`** (`T : struct`, implements **`IComponentPool`**) — dense SoA **`T[]`**, **swap-with-last** removal, reverse map **entity slot → dense row** for O(1) add/remove and tight iteration (`Values` / `Entities` spans). **`Resize(minimumCapacity)`** pre-sizes dense buffers at init; implicit growth during simulation can notify via an optional callback. Resolve via **`Registry.GetPool<T>()`** and **cache** the reference at system init (no pool dictionary lookups in hot loops).
- **Queries (two components):** **`Registry.Query<T1, T2>()`** (`Special.Engine.Ecs.Query`) returns a cached **`Query<T1, T2>`** backed by a **`QueryMatchSet`** that listens to structural pool notifications and maintains a dense **`Entity`** list of entities that have **both** types (O(1) incremental updates on add/remove; **RebuildFromPools** runs once when the query is first registered so entities that already had both components are included). Match order is **insertion order** when the second component of the pair appears. **`RefreshSpans()`** returns **`AlignedQuerySpans<T1,T2>`**—copies into query-owned staging so **`Values1` / `Values2`** are index-aligned (not slices of each pool’s dense arrays, which use different orderings). **`foreach`** uses **`QueryRow`** / **`GetRef`**—same invalidation rules as **`GetRef`**. This is not a full archetype database; parallel job layers must still respect declared read/write access and must not structurally mutate pools while iterating unless single-threaded.
- **Registry:** Slot **generations**, **LIFO free-slot recycling** using **`int[]` + count** (stack semantics; do not use **`List<int>`** for free indices). Private **`Dictionary<Type, IComponentPool>`** maps component types to pools. **`Registry.Destroy`** is **internal**; it recycles the slot only (generation bump + push free index) and **does not** strip pools (see **API Hardening Policy**).
- **Templates / clone:** **`EntityTemplate`** (`Special.Engine.Ecs`) holds ordered apply steps; fluent **`Add<T>(T value)`** / **`Add<T>(Func<T>)`** enqueue **`GetPool<T>().TryAdd`** when **`Registry.Instantiate(template)`** runs (**`CreateEntity`** then **`Apply`**). **`Registry.Clone(source)`** requires **`IsAlive(source)`**, mints a new entity, and walks **`_pools`** once with **`IComponentPool.TryCloneFromTo`** (value copy via **`TryGet`**; if the destination already has that component type it is **removed** then re-added so the result matches the source). **POD-only components** (ints, floats, **`Vector3`**, etc.) should **not** use **`RegisterComponentCloner`**—value copy is correct and faster. For **reference-bearing** structs, register **`RegisterComponentCloner<T>(Func<T,T>)`** (stored by **`Type`**) so clones allocate **new** collections / deep-copied state where needed. In **DEBUG**, cloning a reference-bearing struct **without** a registered cloner logs **`Debug.WriteLine`** (shallow copy still runs). If *many* components need cloners because they embed **`List<>`** / **`Dictionary<>`**, treat that as a smell: prefer **buffers**, **global pools**, or **`Entity`/index handles** instead of reference-heavy on-entity fields—see **`ComponentCloneRegistration`**. Engine **`Components`** are currently all POD. **Class-typed components** are not represented in **`ComponentPool<T>`** (`T : struct` only)—no engine-level deep clone for class-as-component storage yet.
- **CommandBuffer (deferred destroy):** Main-thread queue of entities to tear down. **`Flush(Registry)`** strips pools then **`Registry.Destroy`** for each entry. **`EcsWorld.FlushDeferredDestroys`** runs at end of **`Tick`** after parallel playback.
- **EntityCommandBuffer (parallel structural):** **`EcsWorld`** pre-allocates one buffer per scheduler job index (**`ParallelCommandBuffers`**). During **`IJob.Execute`**, **`JobContext.EntityCommands`** is that job’s buffer: record **`Add<T>`** / **`Remove<T>`** / **`Destroy`** ( **`Add`** boxes the struct). After each topological **`Parallel.For`** layer completes, **`JobScheduler.Dispatch(..., Registry, EntityCommandBuffer[])`** runs **`Playback(Registry)`** on each buffer in job index order—**only** that playback path (plus **`CommandBuffer.Flush`** at tick end) mutates registry slots and pool structure from deferred work; jobs must not call **`TryAdd`**, **`Remove`**, **`CreateEntity`**, or **`Destroy`** on **`Registry`** directly while scheduled in parallel.
- **World / lifecycle:** **`EcsWorld`** owns **`Registry`**, **`CommandBuffer`**, **`EntityCommandBuffer[]`**, **`JobScheduler`**, ordered **`IUpdateSystem`** and **`IFixedUpdateSystem`** lists (no `Entities` façade). **`RequestDestroy`** → **`Commands.RequestDestroy`**. **`Tick(deltaTime)`** fills reusable **`IJob`** scratch lists, runs **`JobScheduler.Dispatch(..., _registry, _parallelCommandBuffers)`** for variable then fixed phases (capped fixed steps), sets **`InterpolationAlpha`**, then **`FlushDeferredDestroys`**. Use **`AddFixedUpdateSystem(snapshot, runFirst: true)`** so **`SnapshotSystem`** runs before physics. No separate store-registration list; pools live on the registry dictionary.
- **System / job:** **`IUpdateSystem`** and **`IFixedUpdateSystem`** extend **`IJob`** (`Special.Engine.Ecs.Jobs`): declare stable per-type **`ReadOnlyComponents`** and **`WriteOnlyComponents`** as **`IReadOnlyList<Type>`** (typically **`static readonly`** arrays built once) and **`Initialize`**; variable **`Update(delta, EntityCommandBuffer?)`** / fixed **`FixedUpdate(delta, EntityCommandBuffer?)`** are invoked via default **`IJob.Execute(in JobContext)`** ( **`JobContext.EntityCommands`** is null when using the two-argument **`Dispatch`** overload without playback). Register with **`EcsWorld.AddSystem`** or **`AddFixedUpdateSystem`**. **`JobScheduler.Dispatch`** uses **`DependencyResolver`**: it builds a **DAG** from those lists (registration order breaks ties for **`i < j`**: writer-after-reader, writer-before-reader, and writer-writer on the same type produce edges **`i → j`**; read-read on the same type does not) and runs **Kahn topological layers**—each layer is **`Parallel.For`** with a reused **`ParallelOptions`** whose **`MaxDegreeOfParallelism`** defaults to **`Environment.ProcessorCount`** (override via scheduler ctor). **`ComponentPool<T>`** is still **not** thread-safe; **`GetRef`** / **`GetRefAtDense`** can escape the pool’s structural discipline, so DEBUG instrumentation only guards **internal** structural paths (`TryAdd`, remove, **`Resize`**, slot-map / dense growth). Do not rely on in-layer job ordering. Parallel layers use **`Parallel.For`** (ThreadPool-backed TPL): do **not** pin systems to threads or spawn long-lived per-system threads—let the pool schedule workers from the tasks submitted each layer; **`MaxDegreeOfParallelism`** is a concurrency ceiling only, not thread assignment. Buffer index **`k`** is the batch slot ( **`IJob`** instance ), not a fixed OS thread.
- **Spatial uniform grid:** **`ISpatialStructure`** defines **`Update(ReadOnlySpan<Entity>, ReadOnlySpan<Vector2>)`** and **`Query(Vector2, float, List<Entity>)`** for broad-phase indices. **`Special.Engine.Spatial.UniformGrid`** implements it with **Head/Next** linked lists in flat arrays: **`int[] Head`** (one entry per cell; **`-1`** means empty) and **`int[] Next`** (span index chain per cell), plus **`Entity[]`** span-index mapping. Update runs two parallel phases: **clear heads** then **populate** via **`Interlocked.Exchange(ref Head[cell], i)`** + **`Next[i] = prev`**. World coverage is half-open from **`(OriginX, OriginY)`** with **`CellsX` / `CellsY`** and **`CellSize`**; out-of-range world coordinates clamp to edge cells. **`UniformGridRebuildSystem`** (**`IFixedUpdateSystem`**, read-only **`Position`**) copies live pool rows into reusable arrays and rebuilds the grid after movement writers.
- **Spatial lifecycle contract:** Grid rebuild occurs during fixed dispatch (after position writers) and is not automatically rerun after **`FlushDeferredDestroys`** at end of tick. Query outputs are **candidates** and consumers must guard with **`Registry.IsAlive(entity)`** (then distance/narrow-phase checks) before expensive collision logic. If a same-tick post-flush-consistent grid is required, explicitly call **`ISpatialStructure.Update`** again after flush.
- **Collision detection contract:** **`CollisionLayerComponent`** (`Layer`, `Mask`) is the collision-layer source of truth. **`Collider`** keeps geometry (`Radius`) and may retain a legacy `LayerMask` field for transition/interoperability, but collision behavior decisions must read `CollisionLayerComponent`. **`CollisionSystem`** is detection/reporting only: it queries **`UniformGrid`** for broad-phase candidates, validates ordered pairs and mask filtering, runs circle-circle narrow phase (`distSq <= (rA + rB)^2`), and writes **`CollisionEvent`** (`EntityA`, `EntityB`, midpoint `ContactPoint`) into thread-safe **`CollisionEventBuffer`**. Resolution is handled downstream by **`CollisionResolverSystem`**, which drains events and queues structural mutations through **`EntityCommandBuffer`** handlers and matrix-dispatches by `CollisionLayer.ToMatrixIndex()`.
- **Debug visualization contract:** **`DebugRenderSystem`** performs a read-only ECS pass and emits `DebugPrimitive` data into shared **`DebugDrawBuffer`**. Systems/jobs never call graphics APIs directly. Renderer-side code consumes `DebugDrawBuffer.AsArray()` during frame render (`OnFrame` / post-update), issues draw calls, then clears the buffer for reuse.
- **Interpolation:** **`Transform`** double-buffers **`Vector3`**; **`SnapshotSystem`** copies **current → previous** at the start of each fixed step. Batch read **`ComponentPool<Transform>.Values`** into **`MathUtils.Interpolate(ReadOnlySpan<Transform>, Span<Vector3>, alpha)`** with **`EcsWorld.InterpolationAlpha`** for cache-friendly render sampling.

## Tick / loop

- **Fixed vs frame:** Simulation should advance on a **fixed timestep**; rendering and interpolation use **variable** wall deltas. See `Special.Engine.Loop.GameLoop` + `GameTime`.
- **Manual:** Rationale and usage examples live under [`.project/`](../../.project/README.md).

## Isometric Rendering Rules

- **Coordinate System:** [Define your system, e.g., X,Y screen coordinates vs. Isometric depth].
- **Sorting:** Entities must be sorted by a `Depth` value calculated from their screen Y-position before draw calls.
- **Batching:** Systems must group draw calls by Texture/Material to minimize state changes.

## Modding Policy

- **Interface-First:** Every game system must implement an interface (e.g., **`IUpdateSystem`** / **`IFixedUpdateSystem`**).
- **Loading:** Mods must load via `AssemblyLoadContext` to allow dynamic unloading.
- **Exposed API:** Only interfaces marked with `[PublicAPI]` attribute are safe for modders to access.

## Multiplayer Policy

- **Server-Authoritative:** The Client sends `InputPackets`. The Server computes the state and broadcasts `StateSnapshots`.
- **Interpolation:** The Client must implement visual interpolation between the last two received snapshots.

## Teardown policy

- **Deferred destruction:** Systems call **`EcsWorld.RequestDestroy(entity)`** (or **`CommandBuffer.RequestDestroy`** on the world’s buffer), which only enqueues.
- **Single flush pipeline:** **`EcsWorld.FlushDeferredDestroys`** → **`CommandBuffer.Flush(Registry)`** → for each queued entity: **`Registry.RemoveFromAllPools`** then **`Registry.Destroy`**. The registry **never** strips pools outside that flush path.

## API Hardening Policy

- **`Registry.Destroy`** is **`internal`**. Only the engine’s flush pipeline may recycle slots after pools are stripped.
- **Systems** must request teardown via **`CommandBuffer.RequestDestroy(entity)`** (typically through **`EcsWorld.RequestDestroy`**, which forwards to the world’s buffer).
- **Direct destruction** of entities (calling slot recycle or skipping the command buffer in ways that leave components attached) is **prohibited** outside that pipeline, to prevent **partial-teardown** state corruption.

## Component storage lookup

- **`Registry`** keeps **`private Dictionary<Type, IComponentPool>`** for type → pool.
- **Mandatory pattern:** systems call **`GetPool<T>()`** once at init and **cache** `ComponentPool<T>` (or `IComponentPool`); **no** `GetPool` / dictionary lookups inside **`Update` / `FixedUpdate` / `OnFixedStep`**.
- **Iteration vs liveness:** Do not treat **`Values`** / **`Entities`** span iteration alone as proof an occupant is still a live entity; use **`Registry.IsAlive`** (or store query APIs) when correctness depends on handle validity—especially if you add **custom **`IComponentPool`** implementations** that must stay consistent with the registry.
